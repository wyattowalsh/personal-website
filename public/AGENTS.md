# public/ — Static Assets

## Structure

```
public/
├── particles/          # TSParticles theme configs
│   ├── dark/          # Dark mode effects
│   └── light/         # Light mode effects
├── *.webp, *.png      # Images (hero images, logos)
├── favicon-*.png      # Favicons (multiple sizes)
├── robots.txt         # Crawler rules
└── sitemap*.xml       # Auto-generated sitemaps
```

## Particle Configurations

Location: `particles/{theme}/{effect}.json`

Effects available: `fireflies`, `galaxy`, `matrix`, `nebula`, `stars`

### Config Structure (TSParticles format)

```json
{
  "autoPlay": true,
  "background": { "color": { "value": "#0d0d1d" } },
  "particles": {
    "number": { "value": 120, "density": { "enable": true, "area": 800 } },
    "color": { "value": ["#ffd700", "#ffa500"] },
    "shape": { "type": "circle" },
    "opacity": { "value": { "min": 0.1, "max": 0.8 }, "animation": { "enable": true } },
    "size": { "value": { "min": 1, "max": 4 } },
    "move": { "enable": true, "speed": 0.5, "direction": "none" },
    "links": { "enable": false }
  },
  "interactivity": {
    "events": { "onHover": { "enable": true, "mode": "grab" } }
  }
}
```

### Adding New Effects

1. Create `particles/dark/{effect}.json` and `particles/light/{effect}.json`
2. Use TSParticles config format (see existing files for examples)
3. Update `components/particles/` to expose new effect

## Images

- Blog hero images referenced in frontmatter: `/image-name.svg`
- Keep images optimized (WebP preferred)
- Use descriptive filenames

## Do Not Modify

- `sitemap*.xml` — Auto-generated by next-sitemap
- `robots.txt` — Usually static
- Favicon files — Update through design workflow
