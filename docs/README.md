# docs

This directory is the standalone Fumadocs workspace for the project documentation site.
It is separate from the main website app and should be run through the pnpm workspace from the repository root.

## Commands

```bash
pnpm --filter docs dev
pnpm --filter docs build
```

## Workspace Layout

- `content/docs/`: authored documentation pages and `meta.json` navigation
- `app/`: Fumadocs app routes, layout config, and search route
- `lib/source.ts`: Fumadocs source loader with the `/docs` base URL
- `source.config.ts`: MDX source definition for the docs content tree
- `public/`: static assets used by the docs site
- `.source/`: generated Fumadocs artifacts, do not edit by hand

## Authoring Notes

- Add or update docs pages under `content/docs/`
- Keep sidebar/navigation changes in sync with `content/docs/meta.json`
- Keep docs aligned with the main site implementation they describe
- Use the docs workspace for internal documentation, not authored blog content

## Boundaries

- Do not edit `.source/`; it is generated by `fumadocs-mdx`
- Validate docs changes with `pnpm --filter docs build`
- If you change content structure or routing, confirm that navigation still resolves under `/docs`
