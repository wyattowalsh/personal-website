## Overview

The developer docs live in the separate `docs/` workspace. It is its own Next.js app, built with **Fumadocs**, and it reads MDX content from `docs/content/docs/`.

## Content Source

`docs/source.config.ts` defines the MDX source:

```ts
import { defineDocs } from "fumadocs-mdx/config";

export const { docs, meta } = defineDocs({});
```

Fumadocs turns that into the generated `@/.source` module used by the app.

## Source Loader

`docs/lib/source.ts` wires Fumadocs into a typed source API:

```ts
export const source = loader({
  baseUrl: "/docs",
  source: createMDXSource(docs, meta),
  icon(icon) {
    // resolves strings like "fa/FaBook" through react-icons
  },
});
```

Key behaviors:

- `baseUrl: "/docs"` fixes the route prefix
- pages and folder metadata are loaded from the generated MDX source
- icon strings from frontmatter and `meta.json` are converted into React elements

## Rendering a Docs Page

The live docs route uses `DocsPage` and injects MDX components like this:

```tsx
import { source } from "@/lib/source";
import {
  DocsPage,
  DocsBody,
  DocsDescription,
  DocsTitle,
} from "fumadocs-ui/page";
import { createRelativeLink } from "fumadocs-ui/mdx";
import { getMDXComponents } from "@/mdx-components";

export default async function Page(props: {
  params: Promise<{ slug?: string[] }>;
}) {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();

  const MDXContent = page.data.body;

  return (
    <DocsPage toc={page.data.toc} full={page.data.full}>
      <DocsTitle>{page.data.title}</DocsTitle>
      <DocsDescription>{page.data.description}</DocsDescription>
      <DocsBody>
        <MDXContent
          components={getMDXComponents({
            a: createRelativeLink(source, page),
          })}
        />
      </DocsBody>
    </DocsPage>
  );
}
```

Important details:

- the page component uses `source.getPage()`
- `DocsPage` receives `toc` and `full`
- relative MDX links are resolved with `createRelativeLink(source, page)`

## Docs Layout

The docs layout imports from `fumadocs-ui/layouts/docs`:

```tsx
import { DocsLayout } from "fumadocs-ui/layouts/docs";
```

That import path matters; `fumadocs-ui/layout` is not what this repo uses.

## Navigation and Ordering

Navigation is filesystem-driven:

- folders become sections
- `.mdx` files become pages
- `meta.json` controls titles, icons, and ordering

Example:

```json
{
  "title": "Architecture",
  "icon": "fa/FaCode",
  "pages": ["overview", "backend", "frontend"]
}
```

## Search and Page Tree

The app relies on Fumadocs' generated source and `DocsLayout`/page components. In practice, the important APIs are:

- `source.getPage(slug)`
- `source.generateParams()`
- `source.pageTree`

The exact search internals are Fumadocs-owned, so when in doubt, treat the local source loader and page wiring as the code you maintain here.

## Adding or Updating a Page

1. Create or edit an `.mdx` file under `docs/content/docs/`
2. Keep frontmatter simple and accurate
3. Update the folder's `meta.json` if ordering needs to change
4. Run the docs app locally

Minimal frontmatter:

```yaml
---
title: My Page
description: What this page is for.
icon: fa/FaFile
---
```

## Local Development

### Start the docs app

```bash
pnpm --filter docs dev
```

### Build the docs app

```bash
pnpm --filter docs build
```

### Start the production docs app

```bash
pnpm --filter docs start
```

### Regenerate the Fumadocs source module

```bash
pnpm --filter docs postinstall
```

That runs `fumadocs-mdx`.

## Build and Deployment Notes

The docs app is a separate workspace.

- `pnpm build` at the repo root builds the **main site**
- `pnpm --filter docs build` builds the **docs app**

Do not assume the root build automatically covers the docs workspace.

## Troubleshooting

### `@/.source` cannot be resolved

Run:

```bash
pnpm --filter docs postinstall
```

### New page does not show up

- check the file path under `docs/content/docs/`
- verify any relevant `meta.json` ordering
- restart the docs dev server if the generated source is stale

### Icon is blank

Check that the icon string uses `set/IconName` format and that the icon set is supported in `docs/lib/source.ts`.
