Blog posts are rendered with the component registry in `mdx-components.tsx`. Everything listed here is available globally inside `content/posts/**/index.mdx` without local imports.

## Built-In Element Overrides

These replace standard HTML output from Markdown with project-styled React components.

### Headings (`h1`-`h4`)

Headings are wrapped with site typography styles and a `HeadingLink` affordance for copying deep links.

- gradient/hover treatment on larger headings
- consistent spacing
- slug-based anchors from `rehype-slug`

```mdx
## A heading with a link target
```

### Paragraphs (`p`)

Paragraphs use the site's prose sizing, spacing, and muted-to-foreground hover treatment.

### Code (`pre`, `code`)

Fenced blocks render through `CodeBlock`.

Current behavior:

- syntax highlighting from `rehype-prism-plus`
- copy-to-clipboard button
- theme-aware block styling

Not currently provided by `CodeBlock`:

- visible language badges
- a dedicated line-number UI

````mdx
```typescript
const greeting = "Hello, world!";
```

Inline `code` is styled separately.
````

### Images (`img`)

Markdown images are upgraded to `next/image` with fixed render dimensions, hover treatment, and an optional caption overlay derived from the `alt` text.

```mdx
![Alt text describing the image](/image.png)
```

### Links (`a`)

Standard Markdown links are handled by the `a` override in `mdx-components.tsx`.

- same-page anchors stay as normal `<a>` tags
- external links open in a new tab and get an icon
- internal links render through Next.js `<Link>`

```mdx
[Internal link](/blog/posts/example)
[External link](https://example.com)
[Anchor link](#section-id)
```

### Tables (`table`)

Tables are wrapped in a horizontally scrollable, styled container.

## Custom MDX Components

### `<Callout>`

Semantic callout box with built-in visual variants.

**Props**

- `type` — `"default" | "info" | "warning" | "error" | "success"`
- `icon` — optional Lucide icon component
- `children`

```mdx
<Callout type="success">
Deployment completed successfully.
</Callout>
```

### `<Details>`

Collapsible disclosure using native `<details>/<summary>`.

**Props**

- `summary` — required summary label
- `children`

```mdx
<Details summary="Why does this work?">
The long explanation goes here.
</Details>
```

### `<CodeFilename>`

Displays a filename strip above a code block.

````mdx
<CodeFilename>src/components/Example.tsx</CodeFilename>
```tsx
export function Example() {
  return <div>Hello</div>;
}
```
````

### `<ScrollableCode>`

Wraps content in `ScrollArea` with a configurable max height.

**Props**

- `maxHeight` — CSS value, default `"400px"`
- `children`

### `<ExternalLink>`

Explicit external CTA-style link component.

```mdx
<ExternalLink href="https://example.com">
Visit Example
</ExternalLink>
```

### `<ClientSideLink>`

Despite the name, this is currently a styled anchor helper implemented with a plain `<a>` tag in `components/ClientSideLink.tsx`.

Use it when you want the component explicitly, but prefer normal Markdown links for most internal navigation.

```mdx
<ClientSideLink href="/blog/posts/example">
Read this post
</ClientSideLink>
```

### `<TagLink>`

Renders a linked blog tag badge.

```mdx
<TagLink tag="TypeScript" />
```

### `<Gist>`

Embeds a GitHub Gist through `GistEmbed`.

```mdx
<Gist id="abc123def456" />
<Gist url="https://gist.github.com/user/abc123def456" file="example.ts" />
```

### `<Separator>`

Decorative separator with a centered logo treatment.

### `<Math>`

Renders KaTeX-backed math. The registry also handles `math`, `math-inline`, and `math-display` names generated by the MDX math pipeline.

```mdx
<Math display>
\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
</Math>
```

### `<Mermaid>`

Client-side Mermaid renderer with zoom/pan/fullscreen controls. Mermaid itself is lazy-loaded in the browser.

```mdx
<Mermaid chart={`
graph LR
  A --> B
  B --> C
`} />
```

## UI Primitives Available in MDX

The registry also exposes a subset of `components/ui/` primitives:

- `<Alert>`
- `<Badge>`
- `<Button>`
- `<Card>` — wrapped with additional spacing and dark-mode classes
- `<Tooltip>` — **root only**
- `<Accordion>`, `<AccordionItem>`, `<AccordionTrigger>`, `<AccordionContent>`
- `<Tabs>`, `<TabsList>`, `<TabsTrigger>`, `<TabsContent>`

### Important limitations

- `AlertTitle` and `AlertDescription` are **not** registered in MDX
- `CardHeader`, `CardTitle`, and `CardContent` are **not** registered in MDX
- `TooltipProvider`, `TooltipTrigger`, and `TooltipContent` are **not** registered in MDX

That means `<Tooltip>` is technically available, but not especially useful on its own in MDX today.

For the full component APIs outside MDX, see [UI Primitives](/docs/reference/ui-primitives).

## Registry Shape

At a high level, `useMDXComponents()` returns:

```typescript
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    h1: ...,
    pre: ({ children }) => <CodeBlock>{children}</CodeBlock>,
    a: ({ href, children }) => /* anchor / external / internal handling */,
    Callout,
    Details,
    Math,
    Mermaid,
    Gist,
    ExternalLink,
    ClientSideLink,
    TagLink,
    // selected ui primitives...
    ...components,
  };
}
```
