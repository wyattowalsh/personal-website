The site currently exposes three custom hooks under `components/hooks/`:

- `useReducedMotion`
- `useWindowSize`
- `useReadingProgress`

All three are browser-facing hooks and should be used from client components.

## `useReducedMotion`

**File:** `components/hooks/useReducedMotion.ts`

This hook tracks the user's `prefers-reduced-motion` media query by using `useSyncExternalStore`.

```typescript
import { useSyncExternalStore } from "react";

const query = "(prefers-reduced-motion: reduce)";

function subscribe(callback: () => void) {
  const mq = window.matchMedia(query);
  mq.addEventListener("change", callback);
  return () => mq.removeEventListener("change", callback);
}

function getSnapshot() {
  return window.matchMedia(query).matches;
}

function getServerSnapshot() {
  return false;
}

export function useReducedMotion(): boolean {
  return useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot);
}
```

### Why this implementation matters

- React gets a stable subscription model instead of an ad hoc `useEffect`/`useState` pair
- the server snapshot is explicit (`false`)
- updates stay in sync with media-query changes

### Usage

```tsx
"use client";

import { motion } from "motion/react";
import { useReducedMotion } from "@/components/hooks/useReducedMotion";

export function AnimatedComponent() {
  const prefersReducedMotion = useReducedMotion();

  return (
    <motion.div
      animate={{ opacity: 1 }}
      transition={{ duration: prefersReducedMotion ? 0.01 : 0.5 }}
    >
      Content
    </motion.div>
  );
}
```

## `useWindowSize`

**File:** `components/hooks/useWindowSize.ts`

This hook tracks the viewport size with an SSR-safe initial state and a built-in debounce.

```typescript
import { useState, useEffect, useRef } from "react";

const DEBOUNCE_MS = 300;

export const useWindowSize = () => {
  const [size, setSize] = useState(() => ({
    width: typeof window !== "undefined" ? window.innerWidth : 0,
    height: typeof window !== "undefined" ? window.innerHeight : 0,
  }));
  const timerId = useRef<ReturnType<typeof setTimeout> | null>(null);

  useEffect(() => {
    const handleResize = () => {
      if (timerId.current !== null) clearTimeout(timerId.current);

      timerId.current = setTimeout(() => {
        setSize({
          width: window.innerWidth,
          height: window.innerHeight,
        });
        timerId.current = null;
      }, DEBOUNCE_MS);
    };

    window.addEventListener("resize", handleResize);
    return () => {
      window.removeEventListener("resize", handleResize);
      if (timerId.current !== null) clearTimeout(timerId.current);
    };
  }, []);

  return size;
};
```

### Current behavior

- initial server-safe fallback is `{ width: 0, height: 0 }`
- resize updates are debounced by `300ms`
- pending timers are cleaned up on unmount

### Usage

```tsx
"use client";

import { useWindowSize } from "@/components/hooks/useWindowSize";

export function ResponsiveComponent() {
  const { width, height } = useWindowSize();

  return (
    <div>
      <p>
        Window: {width}px × {height}px
      </p>
      {width < 768 ? <MobileView /> : <DesktopView />}
    </div>
  );
}
```

## `useReadingProgress`

**File:** `components/hooks/useReadingProgress.ts`

Tracks reading milestones for blog posts and sends analytics events at `25%`, `50%`, `75%`, and `100%`.

```typescript
export function useReadingProgress(slug: string) {
  const firedRef = useRef(new Set<number>());
  const rafId = useRef<number | null>(null);

  useEffect(() => {
    firedRef.current = new Set<number>();
    const fired = firedRef.current;

    function onScroll() {
      if (rafId.current !== null) return;

      rafId.current = requestAnimationFrame(() => {
        rafId.current = null;

        const docHeight =
          document.documentElement.scrollHeight -
          document.documentElement.clientHeight;
        if (docHeight <= 0) return;

        const percent = Math.round((window.scrollY / docHeight) * 100);

        for (const milestone of [25, 50, 75, 100] as const) {
          if (percent >= milestone && !fired.has(milestone)) {
            fired.add(milestone);
            track("reading_progress", { slug, milestone, percent });
          }
        }
      });
    }

    window.addEventListener("scroll", onScroll, { passive: true });
    return () => {
      window.removeEventListener("scroll", onScroll);
      if (rafId.current !== null) cancelAnimationFrame(rafId.current);
    };
  }, [slug]);
}
```

### Behavior

- fires each milestone once per page load
- resets when `slug` changes
- throttles work with `requestAnimationFrame`
- uses a passive scroll listener

## Import Patterns

```typescript
import { useReducedMotion } from "@/components/hooks/useReducedMotion";
import { useWindowSize } from "@/components/hooks/useWindowSize";
import { useReadingProgress } from "@/components/hooks/useReadingProgress";
```

There is no barrel export in `components/hooks/` right now, so import directly from each file.

## Best Practices

1. Add `"use client"` to any component that calls these hooks
2. Respect `useReducedMotion()` whenever you introduce animation
3. Remember that `useWindowSize()` starts at `0x0` during SSR
4. Prefer CSS breakpoints over JS sizing when layout alone is enough
5. Use `useReadingProgress()` for analytics side effects, not rendering state

## Troubleshooting

### `useWindowSize()` looks stale

That may be the `300ms` debounce doing its job. If you need frame-level resize behavior, this hook is probably the wrong tool.

### `useReducedMotion()` always returns `false`

Check the OS/browser reduced-motion preference first. The server snapshot is intentionally `false`, and the real value is read in the browser.
