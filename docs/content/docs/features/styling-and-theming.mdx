## Overview

The website uses a sophisticated hybrid styling engine combining **Tailwind CSS v4** for utility-first development with **SCSS** for complex theming, custom animations, and component-specific styles. This approach provides the rapid development benefits of Tailwind while maintaining the power and flexibility of SCSS for advanced styling needs.

## Architecture

### Core Files

| File | Purpose |
|------|---------|
| `tailwind.config.js` | Tailwind configuration: custom colors, animations, typography |
| `app/globals.scss` | Global styles: CSS variables, layers, prose styling, animations |
| `app/variables.module.scss` | SCSS variables and theme mixins |
| `lib/utils.ts` | `cn()` utility for className composition |

### Styling Philosophy

1. **Tailwind for layout and utilities** — Use Tailwind classes for spacing, layout, typography, and common patterns
2. **SCSS for theming** — CSS custom properties defined in SCSS control colors, shadows, and theme-specific values
3. **Component modules for complex styles** — Individual SCSS modules (e.g., `blogtitle.module.scss`) for advanced component-specific styling

## The `cn()` Utility

From `lib/utils.ts`, the `cn()` function merges Tailwind classes intelligently:

```typescript
import { type ClassValue, clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

**Usage:**

```tsx
import { cn } from '@/lib/utils'

<div className={cn(
  'base-class',
  condition && 'conditional-class',
  'override-class'
)} />
```

`clsx` handles conditionals and arrays, `twMerge` ensures Tailwind class conflicts are resolved (e.g., `p-4` and `p-6` → only `p-6` applies).

## Dark Mode Support

Theme switching via `next-themes`:

```tsx
import { useTheme } from 'next-themes'

const { theme, setTheme, resolvedTheme } = useTheme()
```

- CSS variables update automatically when `.dark` class is toggled on `<html>`
- Server components use `resolvedTheme` to avoid hydration mismatches
- All colors, shadows, and gradients respect theme changes

## Best Practices

1. **Always use `cn()` for className composition** — Never manually concatenate class strings
2. **Prefer Tailwind utilities** — Only use SCSS when Tailwind can't express the style
3. **Use CSS variables for themeable values** — Never hardcode colors or theme-specific values
4. **Keep component SCSS modules minimal** — Extract reusable patterns to `@layer components` in globals.scss
5. **Respect the cascade** — Use `@layer` to control specificity and avoid `!important`

## Common Patterns

### Conditional Classes

```tsx
<div className={cn(
  'base-styles',
  variant === 'primary' && 'bg-primary text-primary-foreground',
  variant === 'secondary' && 'bg-secondary text-secondary-foreground',
  disabled && 'opacity-50 cursor-not-allowed'
)} />
```

### Responsive Utility Stacks

```tsx
<div className={cn(
  'text-sm sm:text-base md:text-lg lg:text-xl',
  'p-2 sm:p-4 md:p-6',
  'flex flex-col sm:flex-row'
)} />
```

### Theme-Aware Backgrounds

```tsx
<div className={cn(
  'bg-background dark:bg-gray-900',
  'text-foreground dark:text-gray-100',
  'border border-border dark:border-gray-700'
)} />
```
