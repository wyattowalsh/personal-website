`components/ui/` is a mix of:

- Radix-backed shadcn-style primitives
- local wrapper components
- a few project-specific components such as `TagPill`

So "UI primitives" is the right mental model, but not every file here is literally a stock shadcn or Radix primitive.

## Import Pattern

```tsx
import { Button, buttonVariants } from "@/components/ui/button";
import { Card, CardHeader, CardTitle, CardContent } from "@/components/ui/card";
```

## Buttons and Inputs

### `Button`

**File:** `components/ui/button.tsx`

Exports:

- `Button`
- `buttonVariants`

Variants:

- `default`
- `destructive`
- `outline`
- `secondary`
- `ghost`
- `link`

Sizes:

- `default`
- `sm`
- `lg`
- `icon`

```tsx
import { Button, buttonVariants } from "@/components/ui/button";

<Button>Click me</Button>
<Button variant="outline">Cancel</Button>
<Button size="icon"><Icon /></Button>

// Reuse the variant styles outside the component
const classes = buttonVariants({ variant: "ghost", size: "sm" });
```

`Button` uses `@radix-ui/react-slot` for its `asChild` behavior.

### `Input`

**File:** `components/ui/input.tsx`

Simple styled input wrapper over native `<input>`.

```tsx
import { Input } from "@/components/ui/input";

<Input type="email" placeholder="email@example.com" />
```

### `Select`

**File:** `components/ui/select.tsx`

Exports:

- `Select`
- `SelectGroup`
- `SelectValue`
- `SelectTrigger`
- `SelectContent`
- `SelectLabel`
- `SelectItem`
- `SelectSeparator`

```tsx
import {
  Select,
  SelectContent,
  SelectGroup,
  SelectItem,
  SelectLabel,
  SelectSeparator,
  SelectTrigger,
  SelectValue,
} from "@/components/ui/select";

<Select value={value} onValueChange={setValue}>
  <SelectTrigger>
    <SelectValue placeholder="Select an option" />
  </SelectTrigger>
  <SelectContent>
    <SelectGroup>
      <SelectLabel>Frameworks</SelectLabel>
      <SelectItem value="next">Next.js</SelectItem>
      <SelectItem value="react">React</SelectItem>
    </SelectGroup>
    <SelectSeparator />
    <SelectItem value="other">Other</SelectItem>
  </SelectContent>
</Select>
```

## Toggle and Tag Components

### `Switch`

**File:** `components/ui/switch.tsx`

Boolean toggle backed by `@radix-ui/react-switch`.

### `TagPill`

**File:** `components/ui/tag-pill.tsx`

Project-specific component that can render as:

- a `Link` when `href` is present
- a `button` when `onClick` is present
- a static `span` otherwise

Variants:

- `default`
- `active`
- `interactive`

```tsx
import { TagPill } from "@/components/ui/tag-pill";

<TagPill tag="TypeScript" />
<TagPill tag="React" href="/blog/tags/React" />
<TagPill tag="Next.js" variant="active" onClick={handleClick} />
```

The current tag route shape is `/blog/tags/[tag]`, so examples should use `/blog/tags/React`, not archive query strings.

## Usage Guidance

1. Prefer these components over ad hoc one-off UI when possible
2. Use `className` for local customization
3. Reuse exported helpers like `buttonVariants` when you need consistent styling outside the component itself
4. Remember that interactive components still need client-component boundaries when used with event handlers or state
