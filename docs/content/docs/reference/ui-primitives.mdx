The w4w.dev website uses **shadcn/ui** components as UI primitives. These are pre-built, accessible components based on Radix UI primitives, styled with Tailwind CSS and located in `components/ui/`.

## Overview

All UI components follow the shadcn/ui pattern:
- ✅ Built on Radix UI headless primitives
- ✅ Styled with Tailwind CSS
- ✅ Fully accessible (ARIA compliant)
- ✅ Customizable with `className` prop
- ✅ Type-safe with TypeScript

**Import pattern:**
```typescript
import { Button } from '@/components/ui/button';
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card';
```

## Form & Input Components

### Button

Versatile button component with multiple variants and sizes.

**File:** `components/ui/button.tsx`

**Variants:**
- `default` — Primary blue button
- `destructive` — Red destructive action
- `outline` — Border with transparent background
- `secondary` — Gray secondary button
- `ghost` — Transparent with hover effect
- `link` — Text link style

**Sizes:**
- `default` — Standard size (h-10)
- `sm` — Small (h-9)
- `lg` — Large (h-11)
- `icon` — Square icon button (h-10 w-10)

**Props:**

```typescript
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'default' | 'destructive' | 'outline' | 'secondary' | 'ghost' | 'link';
  size?: 'default' | 'sm' | 'lg' | 'icon';
  asChild?: boolean; // Render as child component (using Radix Slot)
}
```

**Usage:**

```typescript
import { Button } from '@/components/ui/button';

// Basic button
<Button>Click me</Button>

// Variants
<Button variant="destructive">Delete</Button>
<Button variant="outline">Cancel</Button>
<Button variant="ghost">Dismiss</Button>

// Sizes
<Button size="sm">Small</Button>
<Button size="lg">Large</Button>
<Button size="icon"><Icon /></Button>

// As Link (with asChild)
<Button asChild>
  <Link href="/blog">View Blog</Link>
</Button>

// With custom className
<Button className="mt-4 shadow-lg">Custom Styled</Button>
```

**Radix primitive:** `@radix-ui/react-slot` (for `asChild` prop)

### Toast

Toast notification system.

**Files:**
- `components/ui/toast.tsx` — Toast component
- `components/ui/toaster.tsx` — Toast container

**Usage:**

```typescript
import { Toaster } from '@/components/ui/toaster';
import { useToast } from '@/components/ui/use-toast';

// Add <Toaster /> to root layout
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Toaster />
      </body>
    </html>
  );
}

// Trigger toast in component
export function MyComponent() {
  const { toast } = useToast();

  return (
    <Button onClick={() => {
      toast({
        title: "Success!",
        description: "Post published successfully.",
      });
    }}>
      Publish
    </Button>
  );
}
```

**Radix primitive:** `@radix-ui/react-toast`

## Customization

All components accept `className` prop for customization:

```typescript
// Extend default styles
<Button className="shadow-lg hover:shadow-xl">
  Enhanced Button
</Button>

// Override styles (Tailwind classes with higher specificity)
<Card className="border-2 border-blue-500 rounded-xl">
  Custom Card
</Card>

// Combine with cn() utility
import { cn } from '@/lib/utils';

<Badge className={cn(
  'text-xs',
  isActive && 'bg-green-500',
  !isActive && 'bg-gray-400'
)}>
  Status
</Badge>
```

## Adding New Components

To add a new shadcn/ui component:

```bash
# Using shadcn CLI (if installed)
npx shadcn@latest add [component-name]

# Example
npx shadcn@latest add dialog
npx shadcn@latest add popover
```

Components will be added to `components/ui/`.

## Server vs Client

### Server-Safe Components
Most layout components can be used in server components:
- Card, CardHeader, CardTitle, CardContent
- Badge
- Separator
- Alert

### Client-Only Components
Interactive components require `'use client'`:
- Button (with click handlers)
- Input (with state)
- Checkbox (with state)
- Select (with state)
- Tabs (interactive)
- Accordion (interactive)
- Dropdown Menu (interactive)
- Toast (interactive)
- Tooltip (hover state)

**Usage in server components:**
```typescript
// ✅ OK - no interactivity
<Card>
  <CardTitle>Static Content</CardTitle>
</Card>

// ❌ WRONG - requires client component
<Button onClick={() => alert('clicked')}>
  Click me
</Button>

// ✅ OK - wrap in client component
// ClientButton.tsx
'use client';
export function ClientButton() {
  return <Button onClick={() => alert('clicked')}>Click me</Button>;
}
```
