## **Introduction**

This documentation is a working map of the `w4w.dev` codebase: the main Next.js application, its blog/content pipeline, and the separate Fumadocs workspace that documents it.

Far more than a simple portfolio, the repository combines a content-heavy personal site, a blog with MDX authoring, search and feeds, and a deliberately polished frontend with motion, themed particles, and route-specific UI shells.

**Key Architectural Pillars:**
- **Technology**: Next.js 16, React 19, TypeScript 5, Tailwind CSS 4, `motion/react`, TSParticles, Zod, and Fumadocs.
- **Repository Shape**: pnpm workspace monorepo with the main app at the root and a separate docs app in `docs/`.
- **Backend/Data**: A singleton `BackendService` preprocesses MDX posts, builds search indexes, and serves route-level data needs.
- **API**: Route handlers use shared validation and error middleware from `lib/core.ts`, including correlation IDs and Sentry-aware failure handling.
- **Content**: Blog posts live in `content/posts/` and support MDX, KaTeX-powered math, Mermaid diagrams, tags, and generated feeds.
- **Frontend**: The app uses route-aware layouts, a fixed adaptive header, an animated landing title, and theme-aware particle backgrounds with reduced-motion fallbacks.
- **Components**: A central MDX component registry and `shadcn/ui` primitives provide the building blocks for rich content and interactive UI.
- **Quality**: Strong typing, preprocessing scripts, ESLint flat config, and Vitest tests keep the codebase disciplined.
